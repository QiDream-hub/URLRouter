# URLRouter 特征序列设计文档

**版本**：2.1

---

## 一、概述

特征序列（Feature Sequence）是 URLRouter 路由匹配阶段的核心数据结构。它将段模式编译为一个元组序列，每个元组描述一个**移动操作**和在该位置需要验证的**关键字**。

匹配阶段只执行特征序列，不关心捕获细节，从而实现高性能的路由匹配。

---

## 二、核心概念

### 2.1 操作符分类

段模式中的操作符分为三类，各有不同的编译目标：

| 类别 | 操作符 | 作用 | 特征序列中的处理 |
|------|--------|------|------------------|
| **匹配** | `$'文本'` | 验证关键字 | 产生关键字，参与路由区分 |
| **捕获** | `${}`、`${n}`、`${'c'}` | 提取参数 | 仅移动指针，不参与路由区分 |
| **移动** | `$[n]`、`$[>n]`、`$[<n]`、`$[>'c']`、`$[<'c']`、`$[END]` | 调整指针 | 仅移动指针，不参与路由区分 |

**核心原则**：特征序列只保留匹配所需的信息（移动 + 关键字），捕获操作的参数提取信息被丢弃。

### 2.2 HEAD 与 END

每个段有两个不可变的位置基准：

| 基准 | 定义 | 值 | 性质 |
|------|------|-----|------|
| **HEAD** | 段数组的开始位置 | 0 | 编译时已知的符号标志 |
| **END** | 段数组的结束位置 | 运行时确定的段长度 `len` | 运行时已知的符号标志 |

**重要说明**：
- `HEAD` 和 `END` 都是**符号标志**，不是数值，必须显式出现
- `HEAD` 虽然数值为 0，但语义上代表"段首"这一绝对基准
- `END` 在编译时以符号形式保留，运行时替换为段长度

### 2.3 移动操作的分类

特征元组中的移动操作分为两类：

| 类别 | 基准 | 说明 | 示例 |
|------|------|------|------|
| **常量操作** | HEAD、END 或数值 | 编译时可计算的固定移动 | `(n, kw)`、`(HEAD+n, kw)`、`(END-n, kw)` |
| **动态操作** | 当前指针 | 运行时才能确定的目标位置 | `('c', kw)`、`('<', 'c', kw)` |

---

## 三、基础元组定义

### 3.1 元组类型

| 类别 | 操作符来源 | 基础元组 | 语义 |
|------|-----------|---------|------|
| **常量操作** | `${n}`, `$[>n]` | `(n, NULL)` | 向结尾方向移动 `n` |
| **常量操作** | `$[<n]` | `(-n, NULL)` | 向开头方向移动 `n` |
| **常量操作** | `$[n]` | `(HEAD+n, NULL)` | 移动到 `HEAD + n` |
| **常量操作** | `$[END]`, `${}` | `(END, NULL)` | 移动到段尾 |
| **常量操作** | `$[END-n]` | `(END-n, NULL)` | 移动到 `END - n` |
| **动态操作** | `${'c'}`, `$[>'c']` | `('c', NULL)` | 向结尾方向查找字符 `'c'` |
| **动态操作** | `$[<'c']` | `('<', 'c', NULL)` | 向开头方向查找字符 `'c'` |
| **关键字** | `$'文本'` | `(0, "文本")` | 移动 0 后匹配关键字 |

### 3.2 元组示例

| 操作符 | 基础元组 |
|--------|---------|
| `$'user'` | `(0, "user")` |
| `${2}` | `(2, NULL)` |
| `$[>3]` | `(3, NULL)` |
| `$[<2]` | `(-2, NULL)` |
| `${'='}` | `('=', NULL)` |
| `$[>'=']` | `('=', NULL)` |
| `$[<'=']` | `('<', '=', NULL)` |
| `$[5]` | `(HEAD+5, NULL)` |
| `$[END]` | `(END, NULL)` |
| `${}` | `(END, NULL)` |
| `$[END-4]` | `(END-4, NULL)` |

## 四、编译规则

### 4.1 常量操作相加

同类型的常量操作元组可以数值相加：

| 规则 | 示例 |
|------|------|
| `(n, NULL) + (m, NULL) = (n+m, NULL)` | `(2, NULL) + (3, NULL) = (5, NULL)` |
| `(HEAD+n, NULL) + (m, NULL) = (HEAD+n+m, NULL)` | `(HEAD+2, NULL) + (3, NULL) = (HEAD+5, NULL)` |
| `(END-n, NULL) + (-m, NULL) = (END-n-m, NULL)` | `(END-5, NULL) + (-2, NULL) = (END-7, NULL)` |

**约束**：
- `HEAD` 最终结果必须为 `HEAD + n`，其中 `n ≥ 0`
- `END` 最终结果必须为 `END - n`，其中 `n ≥ 0`
- 不同基准的常量操作不能相加：`(HEAD+2, NULL) + (END-3, NULL)` 无效

### 4.2 动态操作的行为

动态操作**不可相加**，遇到动态操作时：
1. 输出当前 HOLD 的常量操作元组（如有）
2. 动态操作自身成为新的 HOLD

```
输入：${2}${'a'}${3}

状态机：
  IDLE + (2, NULL) → HOLD(2)
  HOLD + ('a', NULL) → 输出 (2, NULL)，HOLD('a')
  扫描结束 → 输出 ('a', NULL)

输出：[(2, NULL), ('a', NULL)]
```

### 4.3 关键字合并

关键字元组 `(0, kw)` 与当前 HOLD 的元组合并，然后输出：

| 规则 | 示例 |
|------|------|
| `(n, NULL) + (0, kw) = (n, kw)` | `(2, NULL) + (0, "key") = (2, "key")` |
| `('c', NULL) + (0, kw) = ('c', kw)` | `('a', NULL) + (0, "key") = ('a', "key")` |
| `(HEAD+n, NULL) + (0, kw) = (HEAD+n, kw)` | `(HEAD+5, NULL) + (0, "key") = (HEAD+5, "key")` |
| `(END-n, NULL) + (0, kw) = (END-n, kw)` | `(END-4, NULL) + (0, "key") = (END-4, "key")` |

```
输入：${2}$'key'

状态机：
  IDLE + (2, NULL) → HOLD(2)
  HOLD + (0, "key") → 输出 (2, "key")

输出：[(2, "key")]
```

```
输入：${'a'}$'key'

状态机：
  IDLE + ('a', NULL) → HOLD('a')
  HOLD + (0, "key") → 输出 ('a', "key")

输出：[('a', "key")]
```

### 4.4 连续关键字的处理

两个 `$'文本'` 操作符可以直接相邻书写，但**不推荐**：

```
输入：$'dd'$'aaa'

状态机：
  (0, "dd") → 输出 (0, "dd")
  (0, "aaa") → 输出 (0, "aaa")

输出：[(0, "dd"), (0, "aaa")]
```

执行时等价于匹配 `"ddaaa"`，但会产生冗余元组。

**建议**：用户应主动合并相邻关键字：
```
// 推荐
$'ddaaa'
```

### 4.5 扫描结束处理

扫描结束时，若 HOLD 中存在元组，则输出：

```
输入：${2}${3}

状态机：
  IDLE + (2, NULL) → HOLD(2)
  HOLD + (3, NULL) → HOLD(5)
  扫描结束 → 输出 (5, NULL)

输出：[(5, NULL)]
```

## 五、编译示例

### 5.1 常量操作相加

```
输入：${1}${1}${1}$'key'

状态机：
  (1, NULL) + (1, NULL) + (1, NULL) = (3, NULL)
  (3, NULL) + (0, "key") = (3, "key")

输出：[(3, "key")]
```

### 5.2 动态操作打断

```
输入：${2}${'a'}${3}$'b'

状态机：
  (2, NULL) → HOLD(2)
  ('a', NULL) → 输出 (2, NULL)，HOLD('a')
  (3, NULL) → 输出 ('a', NULL)，HOLD(3)
  (0, "b") → 输出 (3, "b")

输出：[(2, NULL), ('a', NULL), (3, "b")]
```

### 5.3 绝对移动与常量相加

```
输入：$[5]${2}$'key'

状态机：
  (HEAD+5, NULL) → HOLD(HEAD+5)
  (2, NULL) → HOLD(HEAD+7)
  (0, "key") → 输出 (HEAD+7, "key")

输出：[(HEAD+7, "key")]
```

### 5.4 段尾回退

```
输入：${}$[<4]$'dddd'

状态机：
  (END, NULL) → HOLD(END)
  (-4, NULL) → HOLD(END-4)
  (0, "dddd") → 输出 (END-4, "dddd")

输出：[(END-4, "dddd")]
```

### 5.5 版本号解析

```
输入：/$'v'${'.'}$'.'${}

段模式：$'v'${'.'}$'.'${}

状态机：
  (0, "v") → 输出 (0, "v")
  ('.', NULL) → HOLD('.')
  (0, ".") → 输出 ('.', ".")
  (END, NULL) → HOLD(END)
  扫描结束 → 输出 (END, NULL)

输出：[(0, "v"), ('.', "."), (END, NULL)]
```

### 5.6 查询参数解析

```
输入：/$'search'$'?'${'='}$'='${'&'}$[>1]${'='}$'='${}

段模式：$'search'$'?'${'='}$'='${'&'}$[>1]${'='}$'='${}

状态机：
  (0, "search") → 输出 (0, "search")
  (0, "?") → 输出 (0, "?")
  ('=', NULL) → HOLD('=')
  (0, "=") → 输出 ('=', "=")
  ('&', NULL) → HOLD('&')
  (1, NULL) → 输出 ('&', NULL)，HOLD(1)
  ('=', NULL) → 输出 (1, NULL)，HOLD('=')
  (0, "=") → 输出 ('=', "=")
  (END, NULL) → HOLD(END)
  扫描结束 → 输出 (END, NULL)

输出：[(0, "search"), (0, "?"), ('=', "="), ('&', NULL), (1, NULL), ('=', "="), (END, NULL)]
```

## 六、关键字与唯一性

### 6.1 关键字的作用

关键字是路由分支区分的**唯一依据**：

- 携带关键字（`kw != NULL`）的元组参与路由树的构建和匹配
- 纯移动元组（`kw == NULL`）不参与分支区分

### 6.2 唯一性保证

两条路由的特征序列在**所有携带关键字的元组**上必须不同，否则视为冲突。

元组的类型和参数都参与区分：
- `(3, "key")` 与 `(HEAD+3, "key")` 是不同的元组类型
- `('a', NULL)` 与 `('b', NULL)` 是不同的查找操作
- `(END-4, "key")` 与 `(END-5, "key")` 是不同的偏移量

## 七、数据结构定义

```c
typedef enum {
    FT_CONST_REL_FWD,   // 常量相对向结尾移动: (n, kw)
    FT_CONST_REL_BACK,  // 常量相对向开头移动: (-n, kw)
    FT_CONST_ABS_HEAD,  // 常量绝对位置（基于 HEAD）: (HEAD+n, kw)
    FT_CONST_ABS_END,   // 常量绝对位置（基于 END）: (END-n, kw)
    FT_DYNAMIC_FIND_FWD,// 动态向结尾查找: ('c', kw)
    FT_DYNAMIC_FIND_REV // 动态向开头查找: ('<', 'c', kw)
} feature_type_t;

typedef struct {
    feature_type_t type;
    int value;              // 偏移量或字符 ASCII
    const char *keyword;    // 关键字，可为 NULL
    size_t keyword_len;
} feature_tuple_t;
```

**value 字段语义**：

| type | value 含义 |
|------|-----------|
| `FT_CONST_REL_FWD` | 向结尾移动的距离 |
| `FT_CONST_REL_BACK` | 向开头移动的距离 |
| `FT_CONST_ABS_HEAD` | 从 HEAD 的偏移量（`HEAD + value`），`value ≥ 0` |
| `FT_CONST_ABS_END` | 从 END 的偏移量（`END - value`），`value ≥ 0` |
| `FT_DYNAMIC_FIND_FWD` | 要查找的字符 |
| `FT_DYNAMIC_FIND_REV` | 要查找的字符 |

## 八、与操作符的对应关系

| 操作符 | 基础元组 | 说明 |
|--------|---------|------|
| `$'文本'` | `(0, "文本")` | 关键字元组 |
| `${n}`（后跟关键字） | 合并入后续关键字 | 常量相对移动 + 关键字 |
| `${n}`（末尾） | `(n, NULL)` | 常量相对移动 |
| `${'c'}` | `('c', NULL)` | 动态查找 |
| `${}` | `(END, NULL)` | 常量绝对移动 |
| `$[n]` | `(HEAD+n, NULL)` | 常量绝对移动 |
| `$[>n]` | `(n, NULL)` | 常量相对移动 |
| `$[<n]` | `(-n, NULL)` | 常量相对移动 |
| `$[>'c']` | `('c', NULL)` | 动态查找 |
| `$[<'c']` | `('<', 'c', NULL)` | 动态查找 |
| `$[END]` | `(END, NULL)` | 常量绝对移动 |
| `$[END-n]` | `(END-n, NULL)` | 常量绝对移动 |

## 九、设计原则总结

1. **HEAD 与 END 是符号标志**：必须显式出现，不可省略
2. **常量操作与动态操作分离**：常量操作可相加，动态操作不可相加
3. **关键字驱动**：只有关键字参与路由分支区分
4. **常量相加**：同类型的常量操作元组可以数值相加
5. **动态打断**：动态操作触发当前 HOLD 输出
6. **关键字合并**：关键字与当前 HOLD 元组合并后输出
7. **END 约束**：最终结果必须为 `END - n`，`n ≥ 0`
8. **HEAD 约束**：最终结果必须为 `HEAD + n`，`n ≥ 0`
9. **确定性**：特征序列唯一确定匹配路径
