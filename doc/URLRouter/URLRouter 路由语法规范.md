# URL 路由语法规范

**版本**：2.0

---

## 一、概述

URLRouter 采用基于偏移量的流式段内解析模型。将 URL 按 `/` 切分为段序列，每个段视为字符数组，通过操作指令序列在字符数组上进行模式匹配和数据捕获。

### URL 模型

```
URL = 路径部分

路径部分 = [ "/" ] 段 { "/" 段 }
段 = 除 '/' 外的任意字符序列
```

### 示例

```
URL: /user/profile?format=json&lang=zh

切分结果:
  段 0: "user"
  段 1: "profile?format=json&lang=zh"

URL: /api/v2.0/users/alice
切分结果:
  段 0: "api"
  段 1: "v2.0"
  段 2: "users"
  段 3: "alice"
```

---

## 二、操作符分类

段模式由以下操作符序列组成，操作符按书写顺序执行。

| 类别 | 操作符 | 说明 |
|------|--------|------|
| 匹配操作 | `$'文本'` | 精确匹配固定字符串 |
| 捕获操作 | `${数字}` | 捕获指定长度字符 |
| 捕获操作 | `${'字符'}` | 捕获到指定字符前（不包含该字符） |
| 捕获操作 | `${}` | 捕获到段尾 |
| 移动操作 | `$[位置]` | 绝对跳转 |
| 移动操作 | `$[>偏移]` | 向结尾方向移动 |
| 移动操作 | `$[<偏移]` | 向开头方向移动 |
| 移动操作 | `$[>'字符']` | 向结尾方向查找字符 |
| 移动操作 | `$[<'字符']` | 向开头方向查找字符 |

---

## 三、匹配操作

### 3.1 精确匹配 - `$'文本'`

匹配固定字符串，匹配成功则指针向后移动文本长度。

| 项目 | 说明 |
|------|------|
| 语法 | `$'文本'` |
| 指针变化 | 指针向结尾方向移动文本长度 |
| 产生捕获 | 否 |
| 失败条件 | 文本不匹配或剩余长度不足 |

**示例：**
```
模式：$'user'
段："user"
结果：匹配成功，指针移动到位置 4

模式：$'profile?'
段："profile?format=json"
结果：匹配成功，指针移动到位置 8
```

---

## 四、捕获操作

### 4.1 定长捕获 - `${数字}`

从当前位置捕获指定长度的字符。

| 项目 | 说明 |
|------|------|
| 语法 | `${数字}` |
| 指针变化 | 指针向结尾方向移动指定长度 |
| 产生捕获 | 是 |
| 失败条件 | 剩余长度不足 |

**示例：**
```
模式：${4}
段："2024-03-15"
结果：捕获"2024"，指针移动到位置 4

模式：${3}
段："abc123"
结果：捕获"abc"，指针移动到位置 3
```

### 4.2 捕获到字符 - `${'字符'}`

从当前位置向结尾方向捕获直到遇到指定字符（不包含该字符）。

| 项目 | 说明 |
|------|------|
| 语法 | `${'字符'}` |
| 指针变化 | 指针移动到指定字符位置 |
| 产生捕获 | 是 |
| 失败条件 | 无（未找到时捕获到段尾） |

**示例：**
```
模式：${'='}
段："name=alice"
结果：捕获"name"，指针移动到'='位置

模式：${'&'}
段："q=router&page=2"
结果：捕获"q=router"，指针移动到'&'位置

模式：${'4'}
段："abc4def"
结果：捕获"abc"，指针移动到'4'位置
```

### 4.3 捕获到结尾 - `${}`

从当前位置捕获到段末尾的所有字符。

| 项目 | 说明 |
|------|------|
| 语法 | `${}` |
| 指针变化 | 指针移动到段尾 |
| 产生捕获 | 是 |
| 失败条件 | 永不失败（可能捕获空字符串） |

**示例：**
```
模式：${}
段："alice"
结果：捕获"alice"，指针移动到段尾

模式：${}
段："json&lang=zh"
结果：捕获"json&lang=zh"，指针移动到段尾
```

## 五、移动操作

移动操作用于调整指针位置，不产生捕获。

### 5.1 绝对跳转 - `$[位置]`

将指针移动到段内的绝对位置。

| 项目 | 说明 |
|------|------|
| 语法 | `$[位置]` |
| 位置表达式 | 非负整数 或 `END`（段尾）或 `END - n`（从段尾向开头方向偏移 n） |
| 指针变化 | 指针 = 位置表达式的计算结果 |
| 产生捕获 | 否 |
| 失败条件 | 计算结果超出段范围（小于 0 或大于段长度） |

**位置表达式说明：**
- `END` 表示当前段的长度（即段尾位置）
- `END - n` 表示从段尾向开头方向偏移 n 个字符的位置，n 为非负整数
- 计算结果必须满足 `0 ≤ 位置 ≤ 段长度`

**示例：**
```
模式：$[0]
段："user"
结果：指针移动到位置 0（段开头）

模式：$[5]
段："profile?format=json"
结果：指针移动到位置 5

模式：$[END]
段："alice"
结果：指针移动到位置 5（段尾）

模式：$[END-4]
段："document.pdf"
结果：指针移动到位置 8（段长度 12 减 4）
```

### 5.2 向结尾移动 - `$[>偏移]`

将指针向段结尾方向移动指定距离。

| 项目 | 说明 |
|------|------|
| 语法 | `$[>偏移]` |
| 指针变化 | 指针 += 偏移值 |
| 产生捕获 | 否 |
| 失败条件 | 移动后位置超出段长度 |

**示例：**
```
模式：$[>3]
段："abc123"
当前指针：0
结果：指针移动到位置 3

模式：$[>1]
段："name=alice&age=25"
当前指针：10（在'&'位置）
结果：指针移动到位置 11（跳过'&'）
```

### 5.3 向开头移动 - `$[<偏移]`

将指针向段开头方向移动指定距离。

| 项目 | 说明 |
|------|------|
| 语法 | `$[<偏移]` |
| 指针变化 | 指针 -= 偏移值 |
| 产生捕获 | 否 |
| 失败条件 | 移动后位置小于 0 |

**示例：**
```
模式：$[<4]
段："document.pdf"
当前指针：12（段尾）
结果：指针移动到位置 8（'.'位置）

模式：$[<0]
段："v2.5.1"
当前指针：6
结果：指针移动到位置 0（段开头）
```

### 5.4 向结尾查找字符 - `$[>'字符']`

从当前位置向段结尾方向查找指定字符，将指针移动到该字符位置。

| 项目 | 说明 |
|------|------|
| 语法 | `$[>'字符']` |
| 指针变化 | 指针移动到第一个匹配字符的位置 |
| 产生捕获 | 否 |
| 失败条件 | 未找到指定字符 |

**示例：**
```
模式：$[>'=']
段："name=alice"
当前指针：0
结果：指针移动到位置 4（'='的位置）

模式：$[>'&']
段："q=router&page=2"
当前指针：0
结果：指针移动到位置 10（'&'的位置）

模式：$[>'.']
段："document.pdf"
当前指针：0
结果：指针移动到位置 8（'.'的位置）
```

### 5.5 向开头查找字符 - `$[<'字符']`

从当前位置向段开头方向查找指定字符，将指针移动到该字符位置。

| 项目 | 说明 |
|------|------|
| 语法 | `$[<'字符']` |
| 指针变化 | 指针移动到第一个匹配字符的位置（向开头方向） |
| 产生捕获 | 否 |
| 失败条件 | 未找到指定字符 |

**示例：**
```
模式：$[<'=']
段："name=alice"
当前指针：9（段尾）
结果：指针移动到位置 4（'='的位置）

模式：$[<'&']
段："q=router&page=2"
当前指针：15（段尾）
结果：指针移动到位置 10（'&'的位置）

模式：$[<'.']
段："document.pdf"
当前指针：12（段尾）
结果：指针移动到位置 8（'.'的位置）
```

## 七、语法规则

### 7.1 执行模型

1. 每个段维护一个解析指针，初始值为 0
2. 操作符按书写顺序依次执行
3. 每个操作完成后指针自动更新
4. 所有操作都必须进行边界检查
5. 任何操作失败则整个段匹配失败

### 7.2 边界检查

| 操作符 | 边界条件 |
|--------|----------|
| `$'文本'` | 当前指针 + 文本长度 ≤ 段长度 |
| `${数字}` | 当前指针 + 数字 ≤ 段长度 |
| `${'字符'}` | 无硬性限制（未找到时捕获到段尾） |
| `${}` | 无限制 |
| `$[位置]` | 位置表达式计算结果在 `[0, 段长度]` 范围内 |
| `$[>偏移]` | 当前指针 + 偏移 ≤ 段长度 |
| `$[<偏移]` | 当前指针 - 偏移 ≥ 0 |
| `$[>'字符']` | 无硬性限制（未找到时匹配失败） |
| `$[<'字符']` | 无硬性限制（未找到时匹配失败） |

### 7.3 段匹配结束条件

段匹配成功需同时满足以下两个条件：

1. **操作耗尽**：当前段的所有操作符均已执行完毕
2. **指针对齐**：当前指针位置等于段长度（END）

若操作耗尽但指针未到达段尾，匹配失败。此规则确保模式完整消耗整个段，避免部分匹配导致的歧义。

**示例：**
```
模式：${2}
段："ab"    → 指针移动到 2，等于段长度 → 匹配成功

模式：${2}
段："abc"   → 指针移动到 2，小于段长度 → 匹配失败

模式：${2}$'c'
段："abc"   → 操作耗尽时指针位于 3，等于段长度 → 匹配成功
```

### 7.4 参数捕获

1. 捕获按操作符执行顺序依次存储
2. 每个捕获操作产生一个参数
3. 多个段产生的参数按段顺序连接
4. 参数以（起始指针，长度）形式返回，直接指向原始 URL 数据，不进行复制
5. 空参数通过长度为零表示

---

## 八、语法限制

1. 操作符必须按规范书写
2. 段模式不能为空
3. 路由模式必须以 `/` 开头
4. 偏移值必须是非负整数
5. 捕获长度必须是正整数
6. 捕获字符必须是单个字符，且用单引号包裹，如 `${'='}`
7. 查找字符必须是单个字符，且用单引号包裹，如 `$[>'=']`
8. 位置表达式中的 `END` 后只能跟 `- n` 形式的偏移，n 为非负整数
9. **定长捕获和捕获到字符的区分**：
   - `${数字}` - 定长捕获，如 `${4}` 捕获 4 个字符
   - `${'字符'}` - 捕获到字符，如 `${'='}` 捕获到'='之前
10. **移动操作和捕获操作的区分**：
    - `$[>'字符']` - 查找移动，不捕获
    - `${'字符'}` - 捕获到字符，产生参数

## 九、完整模式示例

### 9.1 基础路由

```
模式：/$'user'/${}
URL:  /user/alice

段 0: $'user' → 匹配"user"
段 1: ${} → 捕获"alice"

捕获结果：["alice"]
```

### 9.2 查询参数

```
模式：/$'search'$'?'$[>'=']$'='${}
URL:  /search?q=router

段 0: $'search' → 匹配"search"
段 1: 
  $'?'        → 匹配"?"
  $[>'=']     → 查找'='，指针移动到'='位置
  $'='        → 匹配"="
  ${}         → 捕获"router"

捕获结果：["router"]
```

### 9.3 多参数查询

```
模式：/$'search'$'?'${'='}$'='${'&'}$[>1]${'='}$'='${}
URL:  /search?q=router&page=2

段 0: $'search' → 匹配"search"
段 1:
  $'?'        → 匹配"?"
  ${'='}      → 捕获"q"
  $'='        → 匹配"="
  ${'&'}      → 捕获"router"
  $[>1]       → 跳过'&'
  ${'='}      → 捕获"page"
  $'='        → 匹配"="
  ${}         → 捕获"2"

捕获结果：["q", "router", "page", "2"]
```

### 9.4 版本号解析

```
模式：/$'api'/$'v'${'.'}$'.'${}
URL:  /api/v2.0

段 0: $'api' → 匹配"api"
段 1:
  $'v'        → 匹配"v"
  ${'.'}      → 捕获"2"
  $'.'        → 匹配"."
  ${}         → 捕获"0"

捕获结果：["2", "0"]
```

### 9.5 文件名和扩展名

```
模式：/$'files'/${'.'}$'.'${}
URL:  /files/document.pdf

段 0: $'files' → 匹配"files"
段 1:
  ${'.'}      → 捕获"document"
  $'.'        → 匹配"."
  ${}         → 捕获"pdf"

捕获结果：["document", "pdf"]
```

### 9.6 定长段限制

```
模式：/${2}/$'key'
URL:  /ab/key

段 0: ${2} → 捕获"ab"，指针移动到位置 2，等于段长度 → 匹配成功
段 1: $'key' → 匹配"key"

捕获结果：["ab"]
```

### 9.7 回溯捕获（先完整后拆分）

```
模式：/$'file'/${}$[0]${'.'}$'.'${}
URL:  /file/document.pdf

段 0: $'file' → 匹配"file"
段 1:
  ${}         → 捕获"document.pdf"
  $[0]        → 移动到位置 0
  ${'.'}      → 捕获"document"
  $'.'        → 匹配"."
  ${}         → 捕获"pdf"

捕获结果：["document.pdf", "document", "pdf"]
```

### 9.8 日期解析

```
模式：/$'logs'/${4}$'-'${2}$'-'${2}
URL:  /logs/2024-03-15

段 0: $'logs' → 匹配"logs"
段 1:
  ${4}        → 捕获"2024"
  $'-'        → 匹配"-"
  ${2}        → 捕获"03"
  $'-'        → 匹配"-"
  ${2}        → 捕获"15"

捕获结果：["2024", "03", "15"]
```

### 9.9 使用查找移动跳过前缀

```
模式：/$'log'$[>'\_']$[>1]${8}
URL:  /log/2024-03-15_ERROR_message

段 0: $'log' → 匹配"log"
段 1:
  $[>'\_']    → 查找'_'，指针移动到'_'位置
  $[>1]       → 跳过'_'
  ${8}        → 捕获 8 个字符"ERROR_me"

捕获结果：["ERROR_me"]
```

### 9.10 从段尾向前查找

```
模式：/$[<'=']${}
URL:  /name=alice

段 0:
  $[<'=']     → 从段尾向前查找'='，指针移动到'='位置
  ${}         → 捕获"=alice"（从'='到段尾）

捕获结果：["=alice"]
```

### 9.11 检查文件扩展名

```
模式：/$[<'\.']$'.pdf'
URL:  /doc.pdf

段 0:
  $[<'\.']    → 从段尾向前查找'.'，指针移动到'.'位置
  $'.pdf'     → 匹配".pdf"

捕获结果：[]  // 仅验证扩展名，不捕获
```

### 9.12 复杂日志格式

```
模式：/$'log'/$[>'\_']$[>1]${8}$[<'\_']${}
URL:  /log/2024-03-15_ERROR_message

段 0: $'log' → 匹配"log"
段 1:
  $[>'\_']    → 查找第一个'_'（在"2024-03-15_ERROR"中的第一个）
  $[>1]       → 跳过'_'
  ${8}        → 捕获"ERROR_me"
  $[<'\_']    → 从当前位置向前查找'_'（在"message"之前）
  ${}         → 捕获"message"

捕获结果：["ERROR_me", "message"]
```

## 十、设计原则

1. **段独立性**：每段解析完全独立，不跨段操作
2. **确定性**：路由模式确定，无可选分支，无条件匹配
3. **指针驱动**：段内维护解析指针，操作符移动指针
4. **最小化**：只保留 9 个正交的基础操作符
5. **无转义**：编译时解析语法，无需运行时转义
6. **段尾对齐**：每个段匹配完成后指针必须位于段尾
7. **职责分离**：移动操作（`$[]`）只移动不捕获，捕获操作（`${}`）负责捕获

## 十一、语法对比

| 旧语法 | 新语法 | 说明 |
|--------|--------|------|
| `${=}` | `${'='}` | 捕获到字符必须用单引号包裹 |
| `${4}` | `${4}` | 定长捕获保持不变 |
| 无 | `$[>'=']` | 新增：向结尾查找字符 |
| 无 | `$[<'=']` | 新增：向开头查找字符 |

**新语法的优势：**
1. 词法分析无歧义，数字和字符可以明确区分
2. 语法更一致，所有字符串/字符都用单引号包裹
3. 移动操作与捕获操作职责分离
4. 支持双向字符查找
5. 支持数字作为捕获字符，如 `${'4'}` 捕获到数字'4'之前
