# URLRouter 语法设计文档

## 一、概述

URLRouter 采用基于偏移量的流式段内解析模型。将 URL 按 `/` 切分为段序列，每个段视为字符数组，通过操作指令序列在字符数组上进行模式匹配和数据捕获。

### URL 模型

```
URL = 路径部分

路径部分 = [ "/" ] 段 { "/" 段 }
段 = 除 '/' 外的任意字符序列
```

### 示例

```
URL: /user/profile?format=json&lang=zh

切分结果:
  段 0: "user"
  段 1: "profile?format=json&lang=zh"

URL: /api/v2.0/users/alice
切分结果:
  段 0: "api"
  段 1: "v2.0"
  段 2: "users"
  段 3: "alice"
```

## 二、段内操作符

段模式由以下操作符序列组成，操作符按书写顺序执行。

### 2.1 精确匹配 - `$'文本'`

匹配固定字符串，匹配成功则指针向后移动文本长度。

| 项目 | 说明 |
|------|------|
| 语法 | `$'文本'` |
| 指针变化 | 指针向结尾方向移动文本长度 |
| 产生捕获 | 否 |
| 失败条件 | 文本不匹配或剩余长度不足 |

**示例：**
```
模式：$'user'
段："user"
结果：匹配成功，指针移动到位置 4

模式：$'profile?'
段："profile?format=json"
结果：匹配成功，指针移动到位置 8
```

### 2.2 定长捕获 - `${数字}`

从当前位置捕获指定长度的字符。

| 项目 | 说明 |
|------|------|
| 语法 | `${数字}` |
| 指针变化 | 指针向结尾方向移动指定长度 |
| 产生捕获 | 是 |
| 失败条件 | 剩余长度不足 |

**示例：**
```
模式：${4}
段："2024-03-15"
结果：捕获"2024"，指针移动到位置 4

模式：${3}
段："abc123"
结果：捕获"abc"，指针移动到位置 3
```

### 2.3 捕获到字符 - `${'字符'}`

从当前位置捕获直到遇到指定字符（不包含该字符）。

| 项目 | 说明 |
|------|------|
| 语法 | `${'字符'}` |
| 指针变化 | 指针移动到指定字符前 |
| 产生捕获 | 是 |
| 失败条件 | 无（未找到时捕获到段尾） |

**示例：**
```
模式：${'='}
段："name=alice"
结果：捕获"name"，指针停在'='前

模式：${'&'}
段："q=router&page=2"
结果：捕获"q=router"，指针停在'&'前

模式：${'4'}
段："abc4def"
结果：捕获"abc"，指针停在'4'前
```

### 2.4 捕获到结尾 - `${}`

从当前位置捕获到段末尾的所有字符。

| 项目 | 说明 |
|------|------|
| 语法 | `${}` |
| 指针变化 | 指针移动到段尾 |
| 产生捕获 | 是 |
| 失败条件 | 永不失败（可能捕获空字符串） |

**示例：**
```
模式：${}
段："alice"
结果：捕获"alice"，指针移动到段尾

模式：${}
段："json&lang=zh"
结果：捕获"json&lang=zh"，指针移动到段尾
```

### 2.5 绝对跳转 - `$[位置]`

将指针移动到段内的绝对位置。

| 项目 | 说明 |
|------|------|
| 语法 | `$[位置]` |
| 位置表达式 | 非负整数 或 `END`（段尾）或 `END - n`（从段尾向开头方向偏移 n） |
| 指针变化 | 指针 = 位置表达式的计算结果 |
| 产生捕获 | 否 |
| 失败条件 | 计算结果超出段范围（小于 0 或大于段长度） |

**位置表达式说明：**
- `END` 表示当前段的长度（即段尾位置）
- `END - n` 表示从段尾向开头方向偏移 n 个字符的位置，n 为非负整数
- 计算结果必须满足 `0 ≤ 位置 ≤ 段长度`

**示例：**
```
模式：$[0]
段："user"
结果：指针移动到位置 0（段开头）

模式：$[5]
段："profile?format=json"
结果：指针移动到位置 5

模式：$[END]
段："alice"
结果：指针移动到位置 5（段尾）

模式：$[END-4]
段："document.pdf"
结果：指针移动到位置 8（段长度 12 减 4）
```

### 2.6 向结尾移动 - `$[>偏移]`

将指针向段结尾方向移动指定距离。

| 项目 | 说明 |
|------|------|
| 语法 | `$[>偏移]` |
| 指针变化 | 指针 += 偏移值 |
| 产生捕获 | 否 |
| 失败条件 | 移动后位置超出段长度 |

**示例：**
```
模式：$[>3]
段："abc123"
当前指针：0
结果：指针移动到位置 3

模式：$[>1]
段："name=alice&age=25"
当前指针：10（在'&'位置）
结果：指针移动到位置 11（跳过'&'）
```

### 2.7 向开头移动 - `$[<偏移]`

将指针向段开头方向移动指定距离。

| 项目 | 说明 |
|------|------|
| 语法 | `$[<偏移]` |
| 指针变化 | 指针 -= 偏移值 |
| 产生捕获 | 否 |
| 失败条件 | 移动后位置小于 0 |

**示例：**
```
模式：$[<4]
段："document.pdf"
当前指针：12（段尾）
结果：指针移动到位置 8（'.'位置）

模式：$[<0]
段："v2.5.1"
当前指针：6
结果：指针移动到位置 0（段开头）
```

## 三、操作符速查表

| 操作符 | 语法 | 消耗字符 | 验证内容 | 产生捕获 |
|--------|------|----------|----------|----------|
| 精确匹配 | `$'文本'` | 是 | 是 | 否 |
| 定长捕获 | `${数字}` | 是 | 否 | 是 |
| 捕获到字符 | `${'字符'}` | 是 | 否 | 是 |
| 捕获到结尾 | `${}` | 是 | 否 | 是 |
| 绝对跳转 | `$[位置]` | 否 | 否 | 否 |
| 向结尾移动 | `$[>偏移]` | 否 | 否 | 否 |
| 向开头移动 | `$[<偏移]` | 否 | 否 | 否 |

**位置表达式说明：**
- `位置` 可以是非负整数、`END` 或 `END - n`
- `END` 表示当前段的长度（段尾位置）
- 计算结果必须为 `[0, 段长度]` 范围内的整数

## 四、语法规则

### 4.1 执行模型

1. 每个段维护一个解析指针，初始值为 0
2. 操作符按书写顺序依次执行
3. 每个操作完成后指针自动更新
4. 所有操作都必须进行边界检查
5. 任何操作失败则整个段匹配失败

### 4.2 边界检查

| 操作符 | 边界条件 |
|--------|----------|
| `$'文本'` | 当前指针 + 文本长度 ≤ 段长度 |
| `${数字}` | 当前指针 + 数字 ≤ 段长度 |
| `${'字符'}` | 无硬性限制（未找到时捕获到段尾） |
| `${}` | 无限制 |
| `$[位置]` | 位置表达式计算结果在 `[0, 段长度]` 范围内 |
| `$[>偏移]` | 当前指针 + 偏移 ≤ 段长度 |
| `$[<偏移]` | 当前指针 - 偏移 ≥ 0 |

### 4.3 段尾检查

特征序列执行完毕后，指针必须位于段尾。此规则确保模式完整消耗整个段，避免部分匹配导致的歧义。

**示例：**
```
模式：${2}
段："ab"
结果：指针移动到位置 2，等于段长度 → 匹配成功

模式：${2}
段："abc"
结果：指针移动到位置 2，小于段长度 → 匹配失败

模式：${2}$'c'
段："abc"
结果：指针移动 2 后验证'c'，再移动 1 到达段尾 → 匹配成功
```

### 4.4 参数捕获

1. 捕获按操作符执行顺序依次存储
2. 每个捕获操作产生一个参数
3. 多个段产生的参数按段顺序连接
4. 参数以（起始指针，长度）形式返回，直接指向原始 URL 数据，不进行复制
5. 空参数通过长度为零表示

## 五、完整模式示例

### 5.1 基础路由

```
模式：/$'user'/${}
URL:  /user/alice

段 0: $'user' → 匹配"user"
段 1: ${} → 捕获"alice"

捕获结果：["alice"]
```

### 5.2 查询参数

```
模式：/$'search'$'?'${'='}$'='${}
URL:  /search?q=router

段 0: $'search' → 匹配"search"
段 1: 
  $'?'   → 匹配"?"
  ${'='} → 捕获"q"（捕获到'='前）
  $'='   → 匹配"="
  ${}    → 捕获"router"

捕获结果：["q", "router"]
```

### 5.3 多参数查询

```
模式：/$'search'$'?'${'='}$'='${'&'}$[>1]${'='}$'='${}
URL:  /search?q=router&page=2

段 0: $'search' → 匹配"search"
段 1:
  $'?'   → 匹配"?"
  ${'='} → 捕获"q"
  $'='   → 匹配"="
  ${'&'} → 捕获"router"（捕获到'&'前）
  $[>1]  → 向结尾移动 1，跳过'&'
  ${'='} → 捕获"page"
  $'='   → 匹配"="
  ${}    → 捕获"2"

捕获结果：["q", "router", "page", "2"]
```

### 5.4 版本号解析

```
模式：/$'api'/$'v'${'.'}$'.'${}
URL:  /api/v2.0

段 0: $'api' → 匹配"api"
段 1:
  $'v'   → 匹配"v"
  ${'.'} → 捕获"2"（捕获到'.'前）
  $'.'   → 匹配"."
  ${}    → 捕获"0"

捕获结果：["2", "0"]
```

### 5.5 文件名和扩展名

```
模式：/$'files'/${'.'}$'.'${}
URL:  /files/document.pdf

段 0: $'files' → 匹配"files"
段 1:
  ${'.'} → 捕获"document"
  $'.'   → 匹配"."
  ${}    → 捕获"pdf"

捕获结果：["document", "pdf"]
```

### 5.6 定长段限制

```
模式：/${2}/$'key'
URL:  /ab/key

段 0: ${2} → 捕获"ab"，指针移动到位置 2，等于段长度 → 匹配成功
段 1: $'key' → 匹配"key"

捕获结果：["ab"]
```

### 5.7 回溯捕获（先完整后拆分）

```
模式：/$'file'/${}$[0]${'.'}$'.'${}
URL:  /file/document.pdf

段 0: $'file' → 匹配"file"
段 1:
  ${}    → 捕获"document.pdf"，指针到段尾
  $[0]   → 移动到位置 0
  ${'.'} → 捕获"document"
  $'.'   → 匹配"."
  ${}    → 捕获"pdf"

捕获结果：["document.pdf", "document", "pdf"]
```

### 5.8 日期解析

```
模式：/$'logs'/${4}$'-'${2}$'-'${2}
URL:  /logs/2024-03-15

段 0: $'logs' → 匹配"logs"
段 1:
  ${4}   → 捕获"2024"
  $'-'   → 匹配"-"
  ${2}   → 捕获"03"
  $'-'   → 匹配"-"
  ${2}   → 捕获"15"

捕获结果：["2024", "03", "15"]
```

### 5.9 段长度限制

```
模式：/${3}${3}${}
URL:  /aaabbbccc

段 0:
  ${3}   → 捕获"aaa"
  ${3}   → 捕获"bbb"
  ${}    → 捕获"ccc"

捕获结果：["aaa", "bbb", "ccc"]
```

### 5.10 复杂日志格式

```
模式：/$'log'$[>10]${8}
URL:  /log/2024-03-15_ERROR_message

段 0: $'log' → 匹配"log"
段 1:
  $[>10] → 向结尾移动 10，跳过"2024-03-15"
  ${8}   → 捕获 8 个字符"ERROR_me"

捕获结果：["ERROR_me"]
```

### 5.11 匹配段尾但不捕获

```
模式：/$'user'$[END]/$'profile'
URL:  /user/alice/profile

段 0: $'user' → 匹配"user"
段 1: $[END] → 移动到段尾，跳过"alice"，不产生捕获
段 2: $'profile' → 匹配"profile"

捕获结果：[]
```

### 5.12 从段尾前定位

```
模式：/$[END-4]${4}
URL:  /2024

段 0:
  $[END-4] → 段长度为 4，移动到位置 0
  ${4}     → 捕获"2024"

捕获结果：["2024"]
```

### 5.13 检查文件扩展名

```
模式：/$[END-4]$'.pdf'
URL:  /doc.pdf

段 0:
  $[END-4] → 段长度为 7，移动到位置 3（'.'的位置）
  $'.pdf'  → 匹配".pdf"，指针移动到位置 7

捕获结果：[]  // 仅验证扩展名，不捕获
```

## 六、设计原则

1. **段独立性**：每段解析完全独立，不跨段操作
2. **确定性**：路由模式确定，无可选分支，无条件匹配
3. **指针驱动**：段内维护解析指针，操作符移动指针
4. **最小化**：只保留 7 个正交的基础操作符
5. **无转义**：编译时解析语法，无需运行时转义
6. **段尾对齐**：每个段匹配完成后指针必须位于段尾

## 七、语法限制

1. 操作符必须按规范书写
2. 段模式不能为空
3. 路由模式必须以 `/` 开头
4. 偏移值必须是非负整数
5. 捕获长度必须是正整数
6. 捕获字符必须是单个字符，且用单引号包裹，如 `${'='}`
7. 位置表达式中的 `END` 后只能跟 `- n` 形式的偏移，n 为非负整数
8. **定长捕获和字符捕获的区分**：
   - `${数字}` - 定长捕获，如 `${4}` 捕获 4 个字符
   - `${'字符'}` - 捕获到字符，如 `${'='}` 捕获到'='之前

## 八、语法对比

| 旧语法 | 新语法 | 说明 |
|--------|--------|------|
| `${=}` | `${'='}` | 捕获到字符必须用单引号包裹 |
| `${4}` | `${4}` | 定长捕获保持不变 |

**新语法的优势：**
1. 词法分析无歧义，数字和字符可以明确区分
2. 语法更一致，所有字符串/字符都用单引号包裹
3. 支持数字作为捕获字符，如 `${'4'}` 捕获到数字'4'之前
