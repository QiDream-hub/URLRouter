# URLRouter 综述文档

## 一、项目定位

URLRouter 是一个为 C 语言环境设计的轻量级 URL 路由库，专用于将 URL 请求路径映射到对应的处理函数，并从 URL 中提取动态参数。

## 二、解决的问题

在 Web 应用、API 服务或任何需要处理 URL 请求的 C 语言项目中，开发者面临以下问题：

1. **路径分发**：如何将不同的 URL 请求交给不同的处理函数
2. **参数提取**：如何从 URL 中提取动态数据（如用户ID、文件名、查询参数）
3. **格式解析**：如何处理各种 URL 格式（RESTful、查询字符串、结构化路径）

URLRouter 提供统一的解决方案，让开发者通过简洁的模式语法描述 URL 格式，自动完成匹配和参数提取。

## 三、核心功能

### 3.1 路由注册

开发者通过模式字符串注册路由规则，每个规则绑定一个处理函数。

```
模式: /$'user'/${}
含义: 匹配 /user/xxx 格式的URL，xxx部分作为参数捕获

模式: /$'search'/$'?'${=}$'='${}
含义: 匹配 /search?key=value 格式的URL，提取键和值
```

### 3.2 路由匹配

当请求到达时，路由器根据 URL 路径找到匹配的处理函数。

```
URL: /user/alice
匹配: /$'user'/${}
结果: 调用对应处理函数，参数为 "alice"
```

### 3.3 参数提取

路由器自动从 URL 中提取动态部分，按顺序返回给调用者。

```
模式: /$'api'/$'v'${.}$'.'${}
URL:  /api/v2.0
提取参数: ["2", "0"]
```

## 四、适用场景

### 4.1 Web 服务器

为不同路径提供不同的处理逻辑。

```
GET /              → 首页处理
GET /about        → 关于页面
GET /user/{id}    → 用户主页
POST /api/login   → 登录接口
```

### 4.2 RESTful API

处理资源导向的 URL 结构。

```
GET    /api/users/{id}        → 获取用户
POST   /api/users             → 创建用户
PUT    /api/users/{id}        → 更新用户
DELETE /api/users/{id}        → 删除用户
```

### 4.3 静态文件服务

解析文件路径和扩展名。

```
GET /static/style.css     → 提取: ["style", "css"]
GET /images/logo.png      → 提取: ["logo", "png"]
```

### 4.4 搜索接口

处理带查询参数的 URL。

```
GET /search?q=keyword&page=2  → 提取: ["q", "keyword", "page", "2"]
```

### 4.5 版本化 API

解析语义化版本号。

```
GET /api/v2.1/users   → 提取: ["2", "1"]
```

## 五、主要特点

### 5.1 轻量级

零外部依赖，仅使用标准 C 库，适合嵌入式系统和资源受限环境。

### 5.2 高性能

采用树形结构存储路由规则，匹配时间复杂度与 URL 段数成正比，与注册路由总数无关。

### 5.3 灵活的表达能力

通过 7 个基础操作符的组合，可描述各种常见的 URL 格式：

- 静态路径匹配
- 动态参数捕获
- 查询字符串解析
- 文件扩展名提取
- 结构化数据解析

### 5.4 HTTP 方法隔离

支持按 HTTP 方法（GET、POST、PUT、DELETE 等）注册不同的处理函数，同一 URL 模式可根据请求方法分发到不同处理器。

### 5.5 确定性匹配

路由匹配结果可预测，不受注册顺序影响，避免隐蔽的路由冲突。

## 六、与其他方案对比

| 对比项 | URLRouter | 硬编码判断 | 正则表达式 |
|--------|-----------|------------|------------|
| 代码复杂度 | 低 | 随路由数量增长 | 中 |
| 匹配性能 | O(段数) | O(路由数) | O(路由数×复杂度) |
| 参数提取 | 自动 | 手动编码 | 需要额外处理 |
| 可维护性 | 高 | 低 | 中 |
| 表达能力 | 针对URL优化 | 任意 | 任意但复杂 |

## 七、使用示例概览

### 基础用法

```c
// 创建路由器
router_t *router = router_create();

// 注册路由
router_register(router, HTTP_GET, "/$'user'/${}", user_handler, NULL);

// 匹配请求
route_params_t params;
route_callback_t handler = router_match(router, HTTP_GET, "/user/alice", &params);

// 调用处理器
if (handler) {
    handler(&params, NULL);
    route_params_free(&params);
}
```

### 模式示例集

| 场景 | 模式 | URL 示例 | 捕获结果 |
|------|------|----------|----------|
| 用户主页 | `/$'user'/${}` | `/user/alice` | `["alice"]` |
| 用户帖子 | `/$'user'/${}/$'posts'/${}` | `/user/alice/posts/123` | `["alice", "123"]` |
| 搜索 | `/$'search'/$'?'${=}$'='${}` | `/search?q=test` | `["q", "test"]` |
| 版本化API | `/$'api'/$'v'${.}$'.'${}/$'users'` | `/api/v2.0/users` | `["2", "0"]` |
| 静态文件 | `/$'static'/${.}$'.'${}` | `/static/logo.png` | `["logo", "png"]` |
| 日期日志 | `/$'logs'/${4}$'-'${2}$'-'${2}` | `/logs/2024-03-15` | `["2024", "03", "15"]` |

## 八、适用项目类型

1. **嵌入式 Web 服务器**：资源有限，需要轻量级方案
2. **IoT 设备 HTTP API**：简单高效的路由需求
3. **C 语言编写的微服务**：需要 HTTP 路由分发
4. **网络库的 URL 分发模块**：作为组件集成
5. **学习项目**：理解路由实现原理

## 九、不适用场景

1. **需要正则表达式匹配**：URLRouter 不使用正则，关注于 URL 特定模式
2. **需要复杂条件路由**：不支持基于请求头、IP 等条件的路由
3. **全功能 Web 框架**：URLRouter 只做路由，不包含模板、会话等功能

## 十、总结

URLRouter 是一个专注、高效、易用的 URL 路由库。它通过简洁的模式语法让开发者用少量代码完成 URL 分发和参数提取，特别适合 C 语言环境下的 Web 开发场景。设计上遵循最小化和确定性原则，在保持轻量的同时提供足够的表达能力满足绝大多数 URL 路由需求。
